# 毕业设计周报
项目阶段：后端服务模块开发 (文件上传与分析)

---

## 一、本周工作总结

本周的核心工作是完成了毕业设计后端模块中文件上传与智能分析服务的功能实现与初步测试。主要围绕文件接收、验证、存储、元数据管理及结构化响应进行了开发。

## 二、具体工作内容

### 1. 核心API端点开发
•   `POST /api/upload` 接口：完成了文件批量上传与分析功能的主接口。支持同时处理多个文件，并可接受多种分析选项参数。

•   `GET /api/paper/{paper_id}` 接口：实现了根据唯一论文ID查询已上传文件元数据与历史分析结果的功能。

•   `DELETE /api/paper/{paper_id}` 接口：实现了删除指定论文（包含其所有上传文件）及其关联数据的清理功能。


### 2. 文件处理逻辑实现
•   安全验证：实现了文件类型（`.pdf`, `.doc`, `.docx`, `.txt`）和后缀名校验，以及文件大小限制（最大50MB）的验证。

•   存储管理：设计了基于唯一`paper_id`的文件目录结构，用于组织用户上传的多个文件，确保了数据隔离性。使用UUID为存储的文件生成唯一名称，避免冲突。

•   异步处理：所有文件读写和验证操作均采用`async/await`异步模式编写，以提高高并发下的I/O效率。


### 3. 元数据与配置化管理
•   `AnalysisOptions` 类：定义了可配置的分析选项（如摘要生成、关键点提取、方法学分析、可视化生成），为后续集成AI分析服务提供了清晰的结构化输入。

•   元数据持久化：实现了`save_paper_metadata`函数，将每次上传任务的`paper_id`、文件列表、时间戳、分析选项等以JSON格式保存，便于跟踪和历史查询。


### 4. 响应结构与错误处理
•   标准化响应：为所有API接口设计了统一的JSON响应格式，包含`success`状态码、操作信息、数据负载和使用的分析选项，提高了前端对接的便利性。

•   异常处理：对文件类型错误、大小超限、文件保存失败、路径不存在、服务器内部错误等场景进行了全面的HTTP异常捕获与友好信息返回。


### 5. 模块化与集成
•   路由设置函数：编写了`setup_upload_routes(app)`函数，能够将本模块的所有路由清晰地集成到主FastAPI应用中，提高了代码的可维护性和可复用性。


## 三、关键技术与代码要点

•   框架：基于FastAPI框架，利用其异步特性、自动数据验证和API文档生成功能。

•   依赖项：使用`UploadFile`, `File`, `Form`处理表单与文件混合数据。

•   文件操作：使用Python标准库`pathlib`和`shutil`进行安全的路径操作和目录删除。

•   唯一标识：采用`uuid`模块为每次上传和分析会话生成全局唯一ID。

•   测试入口：代码底部包含了独立运行测试的入口，可在开发时直接启动UVicorn服务器进行调试。


## 四、下周计划

1.  集成AI服务：将`process_file_analysis`函数中的示例分析逻辑替换为对真实大语言模型API（如项目规划中的DeepSeek或其他模型）的调用，实现真正的文本摘要、要点提取等功能。
2.  数据库集成：考虑将元数据和任务状态持久化到数据库（如SQLite或PostgreSQL），而非仅依赖文件系统。
3.  接口扩展：根据设计文档，开始规划或实现用户管理、分析任务队列管理等其他核心模块的接口。
4.  前端联调：与前端开发同学协作，进行接口联调，根据实际交互需求优化请求与响应格式。
